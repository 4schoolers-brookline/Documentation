    Overview

    Setup

    # make sure you have curl installed

    This portal allows employees, managers, students of 4Schoolers and their parents to have lessons and take exams. Students can have classes and submit their works. Portal uses units based payment system to help students to have classes and teachers to give them.

    Views

    index()

    Validates whether user is logged in as student or not. Redirects to login in case user is not authenticated. Redirects to highlights in case authenticated.

    student_exam_results_api()

    Returns JSON response with requested student's exams results.

    highlights()

    Returns HTML page with the information about logged student's lessons. Calculates how many time left before each lesson.

    req()

    Checks if the request is POST and returns HTML page with request.

    lesson_add()

    Gets form data from request. Creates a new Lesson object and assigns Employee to the lesson. If succesful redirects to lessons .

    lesson_edit()

    Gets form data from request. Edits already existing Lesson object. If succesful redirects to lessons .

    lesson_delete()

    Gets form data from request. Deletes already existing Lesson object. If succesful redirects to lessons .

    to_datetime()

    Takes string variable of date-time as input. Returns datetime formated variable.

    lesson()

    Returns HTML page with the information about particular lesson. Responses to forms request to upload submission. In case of succes creates new Submission object and adds it to Lesson's submissions.

    lessons()

    Returns HTML page with the list of all lessons of logged student.

    directory()

    Returns HTML page with the list of all employees.

    profile()

    Returns HTML page with information about student. Responses to forms request to edit users data. Edits students data.

    logout

    Logouts the user from the portal.

    employee

    Returns HTML page with information about employee.

    password

    Returns HTML page with password changing form. Edits password of the student.

    login

    Checks if User is authenticated. In case of success redirects to user profile. Validates form and logs in user.
